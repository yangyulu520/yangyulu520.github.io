多字段下的excel数据导入

后端
  • java

前言

这几天因为要做一个excel的数据导入，才有了和json和对象，三者之间的恩怨情仇。如果导入的仅仅是几个字段，那很easy，直接通过poi来取字段值，然后塞进对象里就ok了，然而这次要导入的字段有50多个，我敢肯定，这是我碰到的最多字段的表，如果按照常规的方式来做，就相当于要取50次，塞50次，不敢想象这个工作量有多少，所以只能换一种方式。

我采用的方式是：先把excel的数据转换成json数据，然后再把json数据转换成对象。通过这样的方式来达到excel数据的读取，先会叙述过程中遇到的各种问题及处理，最后会贴出自己的代码，毕竟没有实践谈理论，都是扯淡。

excel数据转换成json数据

对于excel的解析，目前普遍采用的是apache的poi。由于之前还做过数据的导出，用的也是这个包，为了以后碰到数据的导入和导出都能够得心应手，就直接把poi封装了一个工具类，放在了我的工具类包中，该类中主要包含excel的读取和写入这两个。

对于excel的读取，比较常规的办法是：

  1. map(代表sheet中的每行数据)中的表头由自己传字段，目的是为了后面json转对象时能根据属性名进行自动映射。一般excel表头都是中文，而我需要映射的话，需要表头为英文
  2. 将excel的数据每个sheet，读取放入到一个List中。
  3. 将刚刚得到的每个sheet的list放入到一个map中，该map的key是excel的每个sheet的名字，value则是每个excel的sheet对应的内容的json串。
  4. 最终返回一个存放了每个sheet内容的map

注：

  • 每个excel的sheet对应内容的json串，就需要用到json解析技术，将list转换成json
  • excel转换成json数据，要考虑到两个特殊的元素，一个是日期，一个是空的单元格。
  • 由于json转换的使用的频繁，我也把json的相关操作，封装进了我的工具类包中，在包中，则对日期和空单元格做了处理。

json数据转换成对象

将excel数据转换成json只是完成了导入的前一半，json再转换成我们需要的对象，则是后一半。在转换的过程中，发现对于json的解析技术有好几种，json-lib、fastjson、jackson、gson。当时差点晕过去，后来在咨询老大及各个博主，群牛下，选择了Jackson作为解析。json串要转换成对象，需要处理好两个东西。
一个是日期的转换，一个是枚举的转换。
json中的日期是串，格式也不统一，而java中的日期一般是java.util.Date.所幸jackson中也考虑到了这个因素，提供了一个设置

来设置好json中的日期格式，进行自动转换。
json中需要映射成枚举属性的值，困扰了我很久。因为，如果说，在对象中，使用了枚举属性，枚举中一般都是用英文来表示其枚举意义。而在excel中存储的一般都是中文。这就涉及到如何将中文的值，映射到英文的枚举上。我百度了好久，都没有找到枚举转换方面遇到的问题的处理。在老大的帮助下包括陆续的咨询，现提供以下三个解决方案：

  1. 枚举属性的set方法，把枚举转换成String，通过对String的判断来插入对应的枚举值

  2. 在构建表结构的时候，不采用枚举，使用int的值来表示，每个值代表一个含义，这样就避免了枚举的转换。

  3. 如果枚举不多的情况下，通过新建一个String的属性接收中文值，然后在转换成对象后，再通过这个属性来设置对应枚举属性的值。

这里推荐第一种的做法。

常规的javabean的设计注意点

  • 一般表的基础字段在开发过程中，难免会出现对于表字段的扩展，一定要注意扩展的字段，之前的数据记录中该字段的值是空的，在使用过程中，会报空的错误。所以，如果需要添加字段，一个是把该字段的的基础数据类型变为对象数据类型，二是在该属性的get方法中添加对于空的处理
