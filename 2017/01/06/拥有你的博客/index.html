拥有你的博客

博客搭建整体流程
博客搭建详细参数

快速搭建好自己的博客，其实网上有很多很多的教程，这里，我再写一遍，更多的是关注在，参考别人的经验时，自己搭建过程中碰到的问题。所以这里直接就先贴出我搭建博客时，参考的这篇教程，下面列出过程中碰到的诸多问题。

github注册

由于之前就注册了gitHub,但是一直没用，也是因为对英文的一点点反抗，就选择了国内提供的 码云作为自己的代码仓库。这次搭建需要使用github page就好好地弄了下。

新建工程

  1. 新建的工程，名字是有格式要求的，必须是 你的用户名.github.io ，默认新建的项目，会自动开启page

貌似github这块，就碰到了这个问题

NodeJs的下载和安装

这个参考文献，基本没啥问题，由于之前已经安装好了nodejs，所以基本这步就是跳过的。

Git的下载和安装

  1. 由于工作需要，Git早早就安装了，但使用命令行来安装hexo时，缺总是无法识别命令，这个坑了我好久，后来才发现，是安装git时的选择问题，一定要选择命令行模式，即

    如果你已经安装了git，请检查是否选择了第二个，还是按照默认安装了第一个。
  2. 新安装的git，是可以复制和通过右键点击粘贴，如果没有，则打开该功能：右击-属性-选项-快速编辑模式

hexo的配置

  1. 关于hexo的配置，首先是安装hexo，对于这个，网上有两三种不同的命令，在尝试后，还是
    ‘’’$ npm install hexo-cli -g’’’
    命令可以。
    然后就是等着hexo安装完成。

  2. 如果之后使用过程中碰到了
    ```python error deployer not found:github ‘’’

    就使用命令 ‘’’ npm install hexo-deployer-git –save ‘’’,然后即可。

  3. 使用hexo，常用命令为：

  • hexo init 初始化hexo
  • hexo g 将md编译成静态页面
  • hexo s 开启服务器，之后可以通过localhost://4000 来访问效果页面
  • hexo clean 清除编译的静态页面
  • hexo d 将本地静态页面上传到git，该命令需要事先配置好_config.yml。–此时不需要用git push
  1. 配置_config.yml来通过 hexo d 上传静态页面，，具体配置参考上面第二个链接

通过以上步骤基本上一个基本的博客搭建，就没有什么问题了。本人也是小菜，也刚接触hexo的方式来搭建博客，希望想搭建的朋友可以顺利成功。

附 主题和域名

主题

每一个博客都有个默认的主题，如果你不喜欢这个主题，可以到网上找一个自己喜欢的主题，进行替换，具体的流程度娘上一大堆。这儿提供一个我自己的主题切换的说明，个人感觉写的很详细 主题切换和插件使用

域名

看着自己博客访问的网址是 xxx.github.io。心里总觉得有一千只草拟马奔腾，太不喜欢了，那就买个域名，绑定自己的网址，以后通过域名来访问自己的博客
。可以通过 GoDaddy来购买域名，看着美元转换成的高额的人民币，对于个人的博客，我还是最终选择了阿里云来购买自己的域名，3元，你值得拥有。
买好了后，需要两步：

  1. 在本地博客文件的source目录下，新建一个CNAME的文件。文件的内容只有 : 你的域名地址(例: www.yangyl520.site)

  2. 在域名设置中，配置域名解析，带www或者不带的都可以解析到你的博客地址，这里不多说，直接贴上我的配置。

到此为止，基本上搭建一个基础博客所涉及到的，都或粗略或详细的说明了下，记录一下两天搭建历程
