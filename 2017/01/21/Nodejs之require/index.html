Nodejs之require()

前端
  • nodejs

系统性的学习又一个服务器端语言nodejs，是没有时间的，所以就学习vue过程中碰到的nodejs相关的内容进行学习和记录

require()

在nodejs中用require来加载模块。require的用法：

当 Node 遇到 require(X) 时，按下面的顺序处理。

  1. 如果 X 是内置模块（比如 require(‘http’）)
    返回该模块。
    不再继续执行.

  2. 如果 X 以 “./“ 或者 “/“ 或者 “../“ 开头

    1. 根据 X 所在的父模块，确定 X 的绝对路径。
    2. 将 X 当成文件，依次查找下面文件，只要其中有一个存在，就返回该文件，不再继续执行。
      X
      X.js
      X.json
      X.node
    3. 将 X 当成目录，依次查找下面文件，只要其中有一个存在，就返回该文件，不再继续执行。
      X/package.json（main字段）
      X/index.js
      X/index.json
      X/index.node

  3. 如果 X 不带路径
    a. 则X在node_modules模块中
    b.依次在该js父目录直至根目录下查找node_modules下的X。

  4. 抛出 “not found”

例：/home/ry/projects/foo.js 执行了 require(‘bar’) ，这属于上述第三种情况

  1. 首先，确定 x 的绝对路径可能是下面这些位置，依次搜索每一个目录。

  2. 搜索时，Node 先将 bar 当成文件名，依次尝试加载下面这些文件，只要有一个成功就返回。

  3. 如果都不成功，说明 bar 可能是目录名，于是依次尝试加载下面这些文件。
