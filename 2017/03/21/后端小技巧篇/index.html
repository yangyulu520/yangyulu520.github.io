后端小技巧篇

后端
  • 后端

mybatis

mybatis的分页列表和根据条件检索的分页列表设计成一个接口，通过mybatis的where搭配if进行判断，同时参数bean中的普通类型转换成对象类型-作用是不传递该参数时无默认值。

mybatis中涉及到字段为mysql的关键字的，更新或者插入，加上`,例：key`

mybatis的resultMap适合bean字段和数据库字段名不一致的映射，或者list集合映射，如果数据库和bean字段都一样，则可以用resultType

mybatis的parameterType可以是map

mybatis的mapper中定义的返回值如果是单个对象，mybatis的xml执行返回会进行判断，如果是一个结果集则自动映射，如果多个，就报错

mybatis中的mappr.java中如果有多个参数，则用注解@Param(“xxx”),这样xml中的语句中parameterType就不起作用了

mybatis的mapper和xml的参数，如果是单个参数，不一定需要名称对应，mybatis会自动解释的

mybatis实际上操作数据库的表是:数据库名.表名.如果不加数据库名，默认使用配置url时的数据库名.分库操作时会涉及到需要指定某个数据库

mybatis的foreach使用：separator表示在每次进行迭代之间以什么符号作为分隔符，例如：

数据库

mysql和Javabean，true和1，FALSE和0会自动对应

数据库表里的数据某一时刻只能某一个数据库线程持有。在并发下，如果涉及表数据的计算，在数据库语句里执行，不会出现数据错误，因为数据库默认更新操作会加锁。在有些特殊情况下，需要在程序里计算，则需要对程序加锁。

对于String主键的数据插入，主键可以使用uuid来生成

对于数据库定时任务(周期存档数据，删除数据)，可以采用mysql事件，事件中调存储过程实现

对于新增表时关联表增加，也可以使用mysql事件来响应

java

bean交给spring接管后，通过继承InitializingBean,DisposableBean这两个接口分别进行bean中资源的分配和释放

后端restful接口设计的原则:模块/页面/表,单表的crud和条件查询使用同一个mapping，通过http状态来区别。

对于页面传递的参数，jsp可以直接通过el表达式来显示

使用spring动态数据源换，数据源的确定必须是在事物外面

消息队列的发送不要放在事物中

当数据错误时，检查绑定的参数是否拼错

spring集成redis和mq等时，注意bean的id不能够一样

思想，一个URL，多个拆分，在service层进行拆分，通过标志，来进行，比如select和selectAll,export导出，都可以用一个action拦截，在service层根据不同参数，调不同的方法

httpclient里封装了很多client访问方法，就不用自己再封装http访问

mvc设计篇—controller层只做调用，不处理业务逻辑，service层处理业务逻辑(调wrap,若一个方法中需要调多个wrap，并且数据源需要切换，则把每个wrap封装一个方法，在方法中进行数据源选择。所有增删改封装一个wrap包裹事物的方法，调不同的mapper)，如果有wrap层，则做事物只读处理，分页包装

maven导出jar包，resource下的配置文件无法加载，需要重新把配置文件放在jar同目录下

程序中，两个double相减，会出现小数点的计算问题，用BigDecimal包装后减。数据库中两个double相减，也会出现精确度的计算问题，类型换成decimal

设计篇：为了达到高通用性，可以定义一个接口，作为方法注入，所有其他实际类继承这个接口，这样，就不必每次修改代码

用httpServletRequest.getParameter接收post请求参数，发送端content-Type必须设置为application/x-www-form-urlencoded；否则会接收不到。

spring中加载系统文件可以直接用resource来获取

java对象作为方法参数，当方法中修改了对象的属性值，原对象也会改变。
