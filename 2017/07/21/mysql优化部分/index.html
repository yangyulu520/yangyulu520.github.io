mysql索引

在有多个索引同时符合的情况下，优先使用颗粒度细的索引，而一般组合索引会比单索引颗粒度更细

所有的范围条件都会导致后续索引失效

mysql开启慢查询日志

注：
slow_query_log //是否打开日志记录
slow_query_log_file //日志存放位置

explain

使用：
explain sql语句

说明：
type=const表示通过索引一次就找到了;key=primary的话，表示使用了主键;
type=all,表示为全表扫描;key=null表示没用到索引;type=ref,因为这时认为是多个匹配行，在联合查询中，一般为REF;
